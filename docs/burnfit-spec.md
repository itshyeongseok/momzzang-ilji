## 0. 분석 요약 — 번핏의 핵심 구조

참고 이미지에서 확인된 사실(추측이 아닌, 화면에 보이는 것):

- **하단 탭 4개**: `투데이` / `커뮤니티` / `분석` / `프로필`. 활성 탭은 파란색 + 아이콘 채움.
- **투데이(홈)**: 상단 주간 날짜 스트립(일~토 + 날짜), 좌상단 스트릭 불꽃, 가운데 업그레이드 알약, 우상단 캘린더 아이콘. 본문은 "계획된 운동" 빈 상태 + `운동 기록하기` CTA + `휴식 모드` 링크, 그 아래 `식단` 카드(0 kcal, 아침/점심/저녁/간식 4칸 + 버튼).
- **프로필 하위 메뉴(투데이 스크롤 영역에도 동일 카드들)**: `운동 목표`, `시간 분석`, `체중`(58kg + 점 4개 추이), `루틴`, `라이브러리`, `프로그램`, `인터벌 타이머`, `화면 설정`.
- **운동 기록 화면**(`6월 28일` 헤더): 상단 `Interval Timer` 바(바로가기), `전체 볼륨` 요약(100kg, 전일대비 빨강), 운동 카드(`1 하체 | 레그 컬` + i버튼 + ⋮), `총 볼륨`, `최근 기록` 버튼, `메모` 인풋, **세트 테이블(세트/kg/회/완료)**, `세트삭제`/`세트추가`, 점선 `운동 추가` 버튼, 하단 메모. 하단 고정바: 운동 시작 전엔 `운동 추가`+`운동 시작`, 시작 후엔 `휴식 타이머(00:53)`+`운동 시간(00:07)`+`운동 완료`.
- **휴식 타이머 모달**: "고생하셨어요! 👏 이제 호흡을 안정시켜볼까요?", 큰 카운트다운(01:00) + 설정값, `다음 운동` 안내, `-10초`/`+10초`, `건너뛰기`(빨강)/`닫기`.
- **운동 선택(라이브러리)**: 검색바, 부위 필터칩(북마크/전체/하체/가슴/등/어깨…), 정렬(`기본순`), 운동 리스트(썸네일+`부위 | 운동명`+북마크+i), 체크박스 다중선택, 하단 `N개의 운동 추가하기`.
- **분석**: `AI 주간/월간 리포트` 카드, `근육 피로도`(앞/뒤 인체 근육맵, 파란 농도=피로), 부위별 볼륨/세트 표, `전월 대비`(평균 운동시간·볼륨 총합), `운동 현황`(시간 + 최근 세션 원형 버블 1:10/0:50…).
- **프로필**: 사용자 카드(이름·KakaoTalk 아바타), 무료/업그레이드 배너, `업적`(운동시간/총 볼륨/총 운동/주간 스트릭 2×2), `3대 중량(1RM)`(TOTAL + 스쿼트/벤치/데드), `최근 신체 정보`(체중/골격근량/체지방률), `운동 기록`(내 루틴/운동 라이브러리/내 메모).

**색 톤**: 강조 파랑 `#3B82F6`계열, 배경은 거의 흰색~연한 회청(`#F2F4F7`), 카드는 순백, 텍스트는 진회/남색(`#1F2937`), 보조 텍스트 회색(`#9CA3AF`). 점선 보더·둥근 알약·큰 라운드(16~20px) 카드가 시그니처.

## 1. 화면(탭) 목록 + 레이아웃·컴포넌트

몸짱일지 현재 탭(홈/운동/식단/습관/신체)을 **번핏 구조에 맞춰 재배치**한다. 번핏의 `커뮤니티`는 서버·계정이 없는 본 앱 철학상 제외, `분석`을 핵심 탭으로 승격.

### 하단 탭 (4개)
| 탭 | 라벨 | 아이콘(이모지/SVG) | 내용 |
|---|---|---|---|
| home | 투데이 | 📅(체크 달력) | 홈 대시보드 |
| log | 운동 | 🏋️(덤벨) | 오늘 운동 기록(세트 입력) |
| stats | 분석 | 📊(막대) | 통계·캘린더·피로도 |
| me | 프로필 | 👤 | 신체·루틴·라이브러리·설정 |

> 식단/습관은 **투데이 홈의 카드**로 흡수(번핏도 식단을 홈 카드로 둠). 별도 탭 제거로 탭 4개 유지(한 손 엄지 도달성).

### 1-1. 투데이(홈) — `#home`
세로 스크롤. 위→아래:
1. **상단바**: 좌 `🔥 N`(연속일), 중앙 `업그레이드` 알약(본 앱에선 → "오늘 한마디/명언" 또는 생략), 우 `📅`(분석 탭의 캘린더로 이동).
2. **주간 날짜 스트립**: 일~토 요일 헤더 + 7일 날짜. 오늘은 파란 원 채움. 기록 있는 날 아래 점(dot). 좌우 스와이프로 주 이동.
3. **오늘 운동 요약 카드**:
   - 기록 없으면 빈 상태(덤벨 아이콘 + "계획된 운동이 없어요" + "지금 바로 운동을 기록해보세요!") + **`운동 기록하기`(primary, 풀폭)** → 운동 탭으로.
   - 기록 있으면 운동명 리스트 + 총 볼륨 요약 + 탭 시 운동 탭.
   - 하단 `휴식 모드` 텍스트 링크(기존 휴식 타이머 오버레이 호출).
4. **식단 카드**: `🍴 식단`, `합계 N kcal` + 진행 바, 아침/점심/저녁/간식 4칸(각 `+`버튼). `>`로 식단 상세(시트).
5. **습관 카드**(기존 습관 흡수): 체크형 습관 도트 + 숫자형 습관 스텝퍼.
6. **체중 카드**: 최근 체중 큰 숫자 + "최근 업데이트: 날짜" + 점 4개 미니 추이. `>`로 신체 시트.
7. (선택) **운동 목표 카드**: "주간 운동 목표 N회 중 M회".

### 1-2. 운동 기록 — `#log`
헤더: `< {월}월 {일}일 >` + 우측 ⋮(메뉴). 본문:
1. **인터벌 타이머 바**(카드): `⏱ 인터벌 타이머` + `바로가기`(설정 시트).
2. **전체 볼륨 요약 바**: `전체 볼륨  {today}kg  {delta vs 최근}` (델타는 +초록/−빨강).
3. **운동 카드**(운동마다 1개):
   - 헤더: `{순번} {부위} | {운동명}` + `ⓘ`(운동 정보) + `⋮`(삭제/대체).
   - `총 볼륨 {kg}` + `최근 기록` 알약(직전 기록 불러오기).
   - `메모` 인풋(운동별 한 줄).
   - **세트 테이블**: 열 `세트 | kg | 회 | 완료`. 각 행: 세트번호(완료 시 막대 아이콘), kg 인풋, 회 인풋, 완료 체크(파랑). 완료 체크 시 자동 휴식 타이머.
   - `— 세트삭제` / `+ 세트추가`.
4. **`+ 운동 추가`** 점선 버튼 → 운동 선택 화면(라이브러리 모달).
5. **세션 메모** 카드: "오늘 운동은 어땠나요? …" placeholder.
6. **하단 고정 액션바**:
   - 시작 전: `운동 추가`(sec) + `운동 시작`(primary).
   - 진행 중: `휴식 타이머 {mm:ss}` | `운동 시간 {mm:ss}` | `운동 완료`(primary).

### 1-3. 운동 선택(라이브러리) — 모달/시트
1. 헤더 `운동 선택하기 ▾`(부위 빠른필터 드롭다운).
2. **검색바** + 우측 스캔 아이콘(본 앱은 검색만).
3. **부위 필터칩 가로 스크롤**: `🔖(북마크) | 전체 | 하체 | 가슴 | 등 | 어깨 | 팔 | 복근 | 유산소`. 선택칩 진남색 채움.
4. **정렬 드롭다운** `기본순 ▾`.
5. **운동 리스트**: 좌 체크박스, 썸네일(없으면 부위색 원+이니셜), `{부위} | {운동명}`, 우측 사용횟수 + 북마크 + `ⓘ`.
6. 하단: 선택칩 미리보기 + `{N}개의 운동 추가하기`(primary). 미선택 시 비활성 "운동을 선택해주세요".

### 1-4. 휴식 타이머 — 오버레이 모달 (기존 재사용·강화)
중앙 모달: 제목 `휴식 타이머` + ⚙(기본시간 설정). 격려 문구("고생하셨어요! 👏 …"). 큰 카운트다운 + 설정값 보조표시. `다음 운동: {운동명}`(없으면 "계획한 운동을 모두 완료했습니다."). `-10초`/`+10초`. 하단 `건너뛰기`(빨강 텍스트) | `닫기`.
> CLAUDE.md 규칙대로 `#view` **바깥 고정 오버레이**로 유지(재렌더 생존).

### 1-5. 분석 — `#stats`
헤더 `분석` + ⚙. 본문:
1. **리포트 카드**: `주간 요약` / `월간 요약`(AI 미사용 → 로컬 집계 텍스트). 번핏의 "AI 리포트" 자리에 규칙기반 요약("이번 주 4회, 총 12.4톤, 가슴 볼륨 최다").
2. **캘린더**(월간): 기록 있는 날 점/색칠. 날짜 탭 시 그날 요약.
3. **부위별 볼륨/세트** 표 또는 막대(가슴/등/하체/어깨/팔/복근).
4. **주간 횟수** 막대(최근 8주).
5. **운동 현황**: 총 시간 + 최근 세션 시간 버블(`1:10 / 0:50 …`).
6. (PRO 영역 대체) **근육 피로도 인체맵**: 우선순위 낮음(SVG 인체 2개+부위 채도). 후순위/선택.

### 1-6. 프로필 — `#me`
1. **사용자 카드**: 이름(고형석) + 한 줄 상태. (아바타는 로컬 이모지/이니셜, 카카오 연동 없음.)
2. **업적 2×2**: 운동 시간 / 총 볼륨 / 총 운동 횟수 / 주간 스트릭(🔥 N주).
3. **3대 중량(1RM)**: TOTAL + 스쿼트/벤치/데드리프트(기록에서 추정 1RM, ⓘ로 산식 설명).
4. **최근 신체 정보 2×3?** → 체중 / 골격근량 / 체지방률(기존 `body` 그대로).
5. **운동 기록 메뉴**: `내 루틴 {N}` / `운동 라이브러리` / `내 메모`.
6. **혜택/설정**: 백업 내보내기·불러오기, 휴식 기본시간, 화면 설정.

## 2. 내비게이션 구조

```
하단 탭(고정, position:fixed bottom)
├─ 투데이(home)  ── 식단 시트, 습관 인라인, 체중 시트, 휴식 모드 오버레이
├─ 운동(log)     ── 운동 선택 모달 ── (세트 입력) ── 휴식 타이머 오버레이
├─ 분석(stats)   ── 캘린더, 부위/주간 차트
└─ 프로필(me)    ── 루틴 시트, 라이브러리, 메모, 설정 시트
```
- 화면 전환은 기존처럼 `renderAll()`이 `#view.innerHTML` 통째 재그림. **탭 상태는 `cur`/localStorage**.
- 모달·시트·휴식타이머·토스트·하단 액션바는 `#view` 바깥 고정 레이어(재렌더 생존, CLAUDE.md 준수).
- 운동 기록 화면은 `log` 탭 자체로 두되, 날짜 헤더(`< 6월 28일 >`)로 다른 날 이동.
- 활성 탭 인디케이터: 아이콘 채움 + 파란 라벨(번핏 동일).

## 3. 라이트 테마 — 디자인 토큰 (정확한 값)

번핏의 밝은 톤을 모사하되 본 앱용으로 확정한 hex. `:root` CSS 변수로 넣고 모든 색을 변수화(하드코딩 금지).

```css
:root{
  /* 배경/표면 */
  --bg:#F2F4F7;            /* 앱 배경(아주 연한 회청) */
  --bg-grad:linear-gradient(180deg,#EAF1FB 0%,#F2F4F7 38%); /* 홈 상단 살짝 하늘 그라데이션(번핏 홈) */
  --surface:#FFFFFF;       /* 카드 */
  --surface2:#F7F8FA;      /* 카드 안 보조 박스(세트행 등) */
  --surface3:#EEF1F5;      /* 비활성/트랙 */

  /* 라인 */
  --line:#E5E8EE;          /* 카드 보더 */
  --line-soft:#EFF1F5;     /* 옅은 구분선 */
  --dash:#C9D2E0;          /* 점선(운동 추가 등) */

  /* 텍스트 */
  --text:#1F2937;          /* 본문/제목(짙은 슬레이트) */
  --text-strong:#111827;   /* 큰 숫자/타이틀 */
  --muted:#6B7280;         /* 보조 설명 */
  --faint:#9CA3AF;         /* placeholder, 단위, 비활성 라벨 */

  /* 강조(번핏 파랑) */
  --accent:#3B82F6;        /* 메인 파랑(버튼/활성탭/링크) */
  --accent-strong:#2563EB; /* 눌림/진한 파랑 */
  --accent-soft:#E8F1FE;   /* 파랑 틴트 배경(칩 선택·알약) */
  --on-accent:#FFFFFF;     /* 파랑 위 글자 */

  /* 보조 강조/상태 */
  --indigo:#6366F1;        /* 리포트/AI 자리 그라데이션 보조 */
  --good:#22C55E;          /* 볼륨 증가(+) */
  --danger:#EF4444;        /* 감소(−)·건너뛰기·삭제 */
  --warn:#F59E0B;          /* 골격근/경고 */
  --gold:#F59E0B;          /* 스트릭 */
  --chip-dark:#2B3442;     /* 북마크칩·식단 + 버튼(번핏 진남색 원) */

  /* 그라데이션 */
  --grad-accent:linear-gradient(135deg,#3B82F6 0%,#2563EB 100%); /* CTA */
  --grad-report:linear-gradient(135deg,#6366F1 0%,#3B82F6 100%); /* 리포트 카드 보더/배지 */

  /* 그림자(라이트에선 은은하게) */
  --shadow-card:0 1px 2px rgba(16,24,40,.04), 0 1px 3px rgba(16,24,40,.06);
```

## 5. 데이터 모델 매핑 (몸짱일지 `DB` 기준)

현재 스키마(CLAUDE.md):
```
workouts:  { 'YYYY-MM-DD': [ {name, sets:[{weight,reps}]} ] }
meals:     { 'YYYY-MM-DD': [ {type, memo, kcal, protein} ] }
habits:    { 'YYYY-MM-DD': { <habitId>: true|number } }
habitDefs: [ {id, name, type, unit?, step?, icon} ]
body:      [ {date, weight, muscle, fat} ]
settings:  { restDefault, autoRest, ... }
```

### 재사용 (변경 없음)
- `meals`, `habits`, `habitDefs`, `body` — 그대로. 식단/습관/신체 카드에 그대로 렌더.
- `settings.restDefault`, `settings.autoRest` — 휴식 타이머에 그대로.

### 최소 변경 (기존 백업 호환 유지 — 항상 기본값 폴백)
| 대상 | 변경 | 호환 |
|---|---|---|
| `workouts[].sets[]` | `{weight,reps}` → `{weight,reps,done?:bool}` | `done` 없으면 미완료로 간주 → 안전 |
| `workouts[]` (운동) | `{name}` → `{name, part?, memo?, durationSec?}` | 부위는 운동명에서 매핑 폴백 |
| `settings` | `+ bestStreak`, `+ goalPerWeek`, `+ intervalPreset` | 없으면 기본값 |

### 신규 추가
| 키 | 형태 | 용도 | 기능 |
|---|---|---|---|
| `exDB` | `[ {id, name, part, unit:'kg', fav?:bool} ]` | 운동 마스터(부위·이름) | F2 라이브러리·필터 |
| `routines` | `[ {id, name, items:[{exId|name, part, sets?}]} ]` | 루틴 분할 저장 | F7 |
| `prs`(선택) | `{ <exId>: {maxWeight, maxVolume, date} }` | 캐시. 없으면 `workouts`에서 계산 | F13 |
| `programs`(선택) | 후순위 | 다주차 프로그램 | F8 |

> **부위 분류(`part`) 표준값**: `하체 | 가슴 | 등 | 어깨 | 팔 | 복근 | 유산소`(번핏 칩과 동일). `exDB.part`로 운동을 분류하고, 기존 `workouts[].name`은 `"{part} | {name}"` 또는 `name`만으로도 동작(폴백).

> **마이그레이션 원칙(CLAUDE.md)**: 구조 변경 시 기존 JSON 백업과 호환을 깨지 않는다. `loadDB` 시 `Object.assign(DEFAULT, saved)`로 새 키 폴백, `sets`에 `done` 없으면 `false` 취급.

## 6. 구현 우선순위 (ROI 순, 단계별)

### 1단계 — 라이트 테마 + 번핏 골격 (체감 가장 큼, CSS 위주)
1. `:root` 라이트 토큰 적용 + 모든 하드코딩 색 변수화(차트 `#4ade80` 등 포함).
2. 하단 탭 4개로 재편(투데이/운동/분석/프로필), 활성=파랑+아이콘 채움.
3. 투데이 홈: 주간 날짜 스트립 + 오늘 운동 요약 카드 + 식단/습관/체중 카드 + `운동 기록하기` primary.
4. 카드/버튼/칩/점선 시그니처 스타일(둥근 18px, 그림자 정책).

### 2단계 — 운동 기록 코어 (앱의 본질)
5. 운동 기록 화면: 운동 카드 + **세트 테이블(세트/kg/회/완료)** + 세트 추가·삭제.
6. `done` 플래그 + 완료 자동 휴식 타이머 연동(F5 강화: 모달 문구·다음 운동·±10초·건너뛰기).
7. 운동 총 볼륨/전체 볼륨 + 전일 대비 델타(F3).
8. 세션 시간 측정(`운동 시작`~`운동 완료`, F9) + 하단 액션바 2-state.

### 3단계 — 운동 DB·라이브러리·루틴
9. `exDB` 시드(부위별 대표 운동 30~50개) + 라이브러리 화면(검색·부위칩·다중선택, F2).
10. `최근 기록` 불러오기(F4).
11. 루틴 저장/불러오기(F7, 로드맵 #3).

### 4단계 — 분석·동기부여
12. 분석 탭: 부위별 볼륨·주간 횟수·캘린더·전월 대비(F11, 로드맵 #4).
13. 프로필: 업적 2×2 + 스트릭 + 1RM 추정(F12, F15) + PR 뱃지(F13, 로드맵 #2).
14. 로컬 규칙기반 "주간/월간 요약"(AI 리포트 자리 대체).

### 5단계 — 선택/후순위
15. 인터벌 타이머(F6), 근육 피로도 인체맵(SVG), 프로그램(F8).
